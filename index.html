The Objective Residue Classification Oracle (ORCO) computes the posterior likelihood of each amino acid class given a set of backbone chemical shifts observed in NMR. Observed shifts are compared against per-residue, per-secondary-structure reference means and standard deviations from the Wishart RefDB using a naive-Gaussian likelihood, with secondary-structure states marginalized out. The reference table is bundled with the page and uses the six backbone atoms HA, CA, CB, CO, N, and HN.

Please cite the following: Manuscript in process.

Input format (SHIFTY):

Paste a SHIFTY-style shift table into the box below. The first line is the column header. Each subsequent line is one residue: NUM AA followed by chemical shifts in the order given by the header. Use 0.00 for missing shifts (any shift exactly equal to 0.00 ppm is treated as absent — this convention is safe for backbone HA/CA/CB/CO/N/HN, which never legitimately read 0.00). Use spaces (not tabs). The AA column may be the one-letter code or ?/X if unknown — it is passed through to the output and not used in classification. The six channels HA, CA, CB, CO, N, and HN are all used by the solver when present. 

#NUM AA  HA    CA    CB    CO     N      HN
1    X   4.05  54.91 32.83 172.14 0.00   0.00
2    ?   4.64  54.42 41.39 175.10 0.00   0.00
3    R   4.33  55.97 30.99 174.82 121.91 8.36
4    V   4.31  60.10 34.32 176.08 118.71 7.99

Paste chemical shifts in SHIFTY format below.

Output: After submission, ORCO displays a preview and provides a downloadable orco_results.txt file containing the top three residue-class predictions per residue.
